# Contributing

CoCoder is early-stage. Keep changes scoped, documented, and aligned with the active Playbook in `cocoder/PRIORITIES.md`.

Before opening a change:

1. Read `AGENTS.md`, `ARCHITECTURE.md`, and the active priority Playbook.
2. Run the relevant package tests.
3. Update docs, ADR indexes, Playbook progress, and session logs when your change affects them.

Use Node 20 LTS and pnpm. Do not commit `local/` files, secrets, run artifacts, or machine-specific paths.
