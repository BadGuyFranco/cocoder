## Item 3 — Sub-Playbook B (adopter onboarding)

**Effort:** real multi-session work (probably 2–4 focused sessions)
**Leverage:** **the marquee remaining v0.1 deliverable.** Without this, CoCoder is unusable by anyone except the founder.

### Scope (from [`2026-05-21-personas-template.plan.md`](./2026-05-21-personas-template.plan.md) — the existing Sub-Playbook B stub)

- **Workspace template** at `templates/workspace-cocoder/` — what `cocoder init` lays down in a new project. Includes `cocoder/AGENTS.md`, `PRIORITIES.md` (stub), `SESSION_LOG.md` (stub), `priorities/`, `tickets/`, `decisions/`, `memory/`, `personas/custom/`, `standards/`, `local/.gitignore`, etc.
- **`cocoder init` command** — copies the template into the target repo. Refuses to run inside the CoCoder install per ADR-0006. `init --merge` for re-running on existing workspaces (idempotent; doesn't overwrite user-edited files without confirmation).
- **`cocoder audit-workspace`** — validates a workspace's `cocoder/` folder against the template structure; reports drift.
- **`cocoder refresh-memory`** — refreshes `cocoder/memory/codebase-map.md` based on current repo state.
- **Full persona library** — extends what Sub-Playbook E pulled forward (Bob + Talia) to the rest: Oscar, Ian, Quinn (script-based), Phil (custom-persona example).
- **Getting-started doc** at `docs/getting-started.md` — the path a stranger follows from `git clone` to first `cocoder launch`.
- **Persona identity regression test** — composed prompts contain persona-identity argv + playbook excerpt verbatim (audit risk: persona drift).

3. **The orchestration loop** is the preferred way to land any test port or bounded code change. Use it via:
   ```sh
   pnpm exec cocoder launch \
     --profile cocoder/profiles/cocoder-dogfood.profile.json \
     --route cocoder/routes/dogfood-port-tests.json \
     --priority-slug v0.1-foundation \
     --workspace-root "/Volumes/NAS LOCAL/CoCoder" \
     --workspace-slug cocoder-dogfood \
     --developer-mode \
     --execute true
   ```
   Edit `cocoder/PRIORITIES.md` "Active task" first to describe what you want done; the orchestration reads that block into the startup-packet.

4. **For non-port work** (e.g., authoring `templates/workspace-cocoder/`), you may need to author a different route + priority-boundary that grants the writer access to `templates/**`. Sub-Playbook E's `dogfood-port-tests.json` is scoped to `packages/core/tests/`; broader scopes need new routes.
