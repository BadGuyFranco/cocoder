## Context

Sub-Playbook C delivers **Oz**: the master orchestration persona's user surface — an HTTP daemon (`packages/oz-daemon`) and React dashboard (`packages/oz-dashboard`) that replace double-click `.command` launches. Founder confirmed Oz is a v0.1 requirement (not v0.2).

The riskiest pieces are **twofold** (same two-invariant pattern as Sub-Playbook B): (1) **daemon security** — Oz can launch and stop processes; a missing CSRF token or DNS-rebinding defense turns Oz into a local privilege-escalation vector; (2) **workspace-registry survival** — Oz reads/writes `local/workspaces.json`; if tokenized paths do not round-trip across Syncthing-synced machines, Oz is unusable on the founder's multi-machine setup and Master Refine P-R3 (recovery test) fails.

### Tasks — Security (C-Solve-1): one test per invariant

- [x] **C-S1** **Bind `127.0.0.1` only** (invariant 1). **Proven:** PR #36 → `7b5cb60` — `packages/core/tests/oz-security-bind.test.mjs`
  - **Implementation shape:** daemon config rejects `host: '0.0.0.0'` / `'::'`; default bind `127.0.0.1`.
  - **Test shape:** `packages/oz-daemon/tests/security-bind.test.mjs` (or equivalent): start test server with explicit `0.0.0.0` config → expect throw/non-zero exit; start with default → assert listening address is `127.0.0.1` via `server.address()` or port-probe that refuses non-loopback connection.

- [x] **C-S2** **Per-install session token** (invariant 2). **Proven:** PR #36 → `7b5cb60` — `packages/core/tests/oz-security-auth.test.mjs`
  - **Implementation shape:** on first start, generate random token → write `<CoCoder>/local/secrets/oz-token` mode `0600`; all `POST`/`PUT`/`DELETE` require `Authorization: Bearer <token>`.
  - **Test shape:** `security-auth.test.mjs`: (a) missing header → 401; (b) wrong token → 401; (c) valid token → 200 on health/settings GET; (d) token file mode `0600` asserted where platform allows.

- [x] **C-S3** **Origin/Host header rejection** (invariant 3 — DNS-rebinding defense). **Proven:** PR #37 → `0df3cf8` — `packages/core/tests/oz-security-origin-host.test.mjs`
  - **Implementation shape:** middleware allowlist `localhost:<port>`, `127.0.0.1:<port>` only.
  - **Test shape:** `security-origin-host.test.mjs`: request with `Host: evil.example` → 403; `Origin: http://evil.example` → 403; legitimate `Host: 127.0.0.1:<port>` → pass through.

- [x] **C-S4** **CSRF token on state-changing requests** (invariant 4). **Proven:** PR #37 → `0df3cf8` — `packages/core/tests/oz-security-csrf.test.mjs`
  - **Implementation shape:** issue CSRF token on dashboard bootstrap endpoint or embedded in initial HTML; require matching header/cookie on `POST`/`PUT`/`DELETE` even when Bearer token present.
  - **Test shape:** `security-csrf.test.mjs`: valid Bearer + missing CSRF → 403; valid Bearer + valid CSRF → success on fixture `POST /runs` stub.

- [x] **C-S5** **Settings never return secret values** (invariant 5). **Proven:** PR #38 → `1e5fc6f` — `packages/core/tests/oz-security-settings-redaction.test.mjs`
  - **Implementation shape:** `GET /settings` serializes config with secret refs only (`ref:env:...`, `${env:...}` literals); never calls `resolveSecrets:true`.
  - **Test shape:** `security-settings-redaction.test.mjs`: seed fixture install config with `${env:OPENAI_API_KEY}`; GET response must not contain resolved env value; must contain ref form; PUT with inline secret rejected or normalized to ref.

- [x] **C-S6** **Audit log append on launch/stop** (invariant 6). **Proven:** PR #40 → `9a09575` — `packages/core/tests/oz-security-audit-append.test.mjs`
  - **Implementation shape:** every launch/stop handler appends one JSON line to `<CoCoder>/local/audit/oz-actions.jsonl`; create `local/audit/` if missing; document rotation policy in `packages/oz-daemon/README.md` (size/time threshold, manual archive — no automatic rotation code required in v0.1).
  - **Test shape:** `security-audit-append.test.mjs`: invoke launch/stop handler in test harness → file exists; line count increments; each line is valid JSON with `timestamp`, `action`, `workspaceId`, `runId`, `outcome` fields.

- [x] **C-S7** **No shell-string interpolation** (invariant 7). **Proven:** PR #38 → `1e5fc6f` — `packages/core/tests/oz-security-spawn.test.mjs`
  - **Implementation shape:** all subprocess spawns use `spawn(cmd, argv[], { shell: false })`; workspace paths passed as argv elements only.
  - **Test shape:** `security-spawn-argv.test.mjs`: (a) static grep/lint over `packages/oz-daemon/src/` forbids `exec(`, `execSync(`, `` spawn(…, { shell: true }) ``; (b) integration fixture launches with workspace path containing spaces → still succeeds (would fail under shell interpolation).
