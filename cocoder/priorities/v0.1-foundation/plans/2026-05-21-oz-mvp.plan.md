## Context

Sub-Playbook C delivers **Oz**: the master orchestration persona's user surface — an HTTP daemon (`packages/oz-daemon`) and React dashboard (`packages/oz-dashboard`) that replace double-click `.command` launches. Founder confirmed Oz is a v0.1 requirement (not v0.2).

The riskiest piece is **daemon security**, not UI polish. Oz can launch and stop processes; a missing CSRF token or DNS-rebinding defense turns Oz into a local privilege-escalation vector that a malicious npm script or browser tab could exploit. Security gets its own Solve.

## Witness

*Detailed Witness at Activation.* High-level: Oz inherits the evidence model from CoBuilder's debugger; the daemon is a thin HTTP layer over `packages/core` operations; the dashboard is a Vite + React app consuming Zod-typed fetch clients generated from `packages/schemas`.

### Objective

Ship `cocoder oz` command that starts a localhost daemon + serves a dashboard with: workspace list, priority launch button per workspace, run status feed, model-map settings, Run Inspector (read-only debugger evidence). Replaces double-click launches for priorities; keeps CLI fallback.

### Scope (locked v0.1 MVP)

**In:** Daemon (`packages/oz-daemon`), dashboard (`packages/oz-dashboard`), 4 pages (Workspaces, Priorities, Runs, Settings), Run Inspector view, concurrency indicator per priority, security hardening per ARCHITECTURE.md, audit log.

**Out:** Hosted/multi-user, codebase visualization, doc-freshness panel (v0.2), persona marketplace, mobile UI, telemetry.

## Solve

**Riskiest piece:** Daemon security hardening.

### Tasks

- [ ] **C-S1** Daemon binds `127.0.0.1` only; integration test asserts `0.0.0.0` binding is rejected
- [ ] **C-S2** Session token generated at first start, stored in `<CoCoder>/local/secrets/oz-token` (mode 0600); all `POST`/`DELETE` endpoints require `Authorization: Bearer <token>`
- [ ] **C-S3** Origin/Host check middleware rejects requests with `Origin` or `Host` not in `localhost:<port>` / `127.0.0.1:<port>`; integration test simulates DNS-rebinding attempt
- [ ] **C-S4** CSRF token issued on dashboard load; required on state-changing endpoints; integration test fails missing-CSRF
- [ ] **C-S5** Settings endpoint serialization strips secret values; round-trip test asserts no secret values appear in `GET /settings` response
- [ ] **C-S6** All launch/stop actions write to `<CoCoder>/local/audit/oz-actions.jsonl`; rotation policy documented
- [ ] **C-S7** No shell-string interpolation: every spawn uses argv array; lint rule + test fixture proves it

**Pass threshold:** All seven security tests green; manual penetration checklist (`docs/oz-security-checklist.md`) reviewed.

**Checkpoint:** [ ] Oz daemon security proven.
