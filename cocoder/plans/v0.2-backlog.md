# v0.2 Backlog — Consolidated deferred items

**Created:** 2026-05-21 | **Updated:** 2026-05-21
**Type:** Backlog (reference file; not a Playbook)
**Status:** Capture-only — promote items to a priority when ready

> Items deferred out of v0.1 during the Master Playbook + Sub-Playbook A–D scoping pass. This file is a holding pen, not a commitment. When a v0.2 priority opens, items move into `priorities/v0.2-*/plans/`.

## How this file works

- **Append-only during v0.1.** Edit only to add new deferrals discovered while executing Sub-Playbooks A–D.
- **Each item must capture WHY it was deferred** (so future-us doesn't relitigate the same trade-off).
- **No prioritization here.** Ordering happens when v0.2 Witness begins.
- **Not a Playbook.** No WISER phases, no Final Check. When promoted, the receiving Playbook will Witness the relevant rows.

### From Sub-Playbook F (Structural cleanup; added 2026-05-23 post-review)

> The 2026-05-23 "Thermo-Nuclear Code Quality Review" surfaced 9+ structural debt findings. Three were B-blocking and landed inside Sub-Playbook F (FB-1/FB-2/FB-3). The six items below were ruled real-debt-but-NOT-B-blocker and queued here per F's Decision Log. **Most or all of these should be folded into a single "v0.2-architectural" priority.**

| Item | Why deferred from v0.1 |
|---|---|
| `packages/core/lib/launch.mjs` 5-module decomposition (lifecycle / preflight / prompt-builder / script-templates / tmux) | 1,882-line god module. Real debt, but Sub-Playbook B doesn't extend `launch.mjs`, so it isn't a B blocker. Pulling it into v0.1 risks Item 3 slipping. ADR-0004 ("extract verbatim first, refactor later") explicitly anticipated this timing. |
| `packages/core/lib/ledger.mjs` split (extract `job-result-validation.mjs` pure validators; extract `run-finalize.mjs` orchestration pipeline; leave `ledger.mjs` as run-record I/O only) | 1,010-line module mixing layers; `dispatch.mjs:35` imports validation from `ledger.mjs` (feature leak). Not a B blocker; defer alongside the launch decomposition. |
| `packages/core/tests/launch.test.mjs` split (4-5 focused test files + shared fixture layer at `tests/helpers/orchestration-fixtures.mjs`) | 2,722-line test file with 55 tests + duplicate fixture builders. Important nuance: this was ported VERBATIM from CoBuilder via PR #3 (E2.2e.5) for behavior-preservation parity per ADR-0004 — split it opportunistically when adding new launch tests rather than as a green-field refactor. |
| `packages/cocoder-cli/src/cli.ts` identity decision (real shape with path resolution + env normalization + typed error shaping, OR collapse to a documented bin symlink until it does real work) | Currently 22-line `spawn` passthrough — ADR-0003's "typed public boundary" rationale is aspirational. | Inline prompt prose extraction (`launch.mjs renderLanePrompt` lines ~999–1151, ~150 lines of policy text with `startupMode === 'lead'` branches) → shipped prompt fragments under `cocoder/personas/prompts/shared/` | Belongs in shipped prompt fragments, not orchestration code. | `renderAttachAddedLanesScript` (~130 lines iTerm AppleScript inside `launch.mjs`) → strategy interface at `packages/core/lib/terminal/attach-strategies.mjs` | macOS/iTerm is an implementation detail of one attach strategy. v0.2 adapter-extensibility (per ADR draft) needs this boundary anyway. Fold into the same v0.2 priority. |
| Full Zod migration of orchestration contracts (`packages/core/contracts/*.json` hand-DSL → `packages/schemas/src/contracts/` Zod, emit JSON Schema, replace `contracts.mjs validateInstance` with AJV) | F's FB-3 patches the immediate correctness gap (`matchesType` honors `enum`). 3. Delete promoted rows from this file (kept lean)
