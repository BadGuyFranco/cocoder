# Glossary

Canonical vocabulary for CoCoder. This page summarizes live terms and links to the owning architecture
or decision surface when one exists.

This is the install-owned glossary for CoCoder framework terms. Each onboarded workspace additionally ships
a per-repo domain glossary at its own `cocoder/glossary.md` for that product's terms-of-art.

| Term | Definition | Owner |
|---|---|---|
| CoCoder | The AI coding orchestration system: engine, dashboard, runner, governance, and shipped persona/Play base. In this repo, `CoCoder/` is also the install root and the dogfood primary root. | [ARCHITECTURE.md](../ARCHITECTURE.md) |
| `cocoder/` | A tracked workspace governance directory. In this dogfood repo, `CoCoder/cocoder/` is CoCoder's own workspace governance; in an adopted repo, `[repo]/cocoder/` has the same structure for that repo. | [ADR-0008](../cocoder/decisions/0008-repository-topology.md) |
| Domain glossary | A per-workspace surface for a repo's product/domain terms-of-art, distinct from this engine glossary; one canonical gloss + owner link per term. | [ADR-0039](../cocoder/decisions/0039-domain-glossary-deliverable.md) |
| Install root | The CoCoder engine checkout containing `packages/`, `docs/`, `templates/`, `scripts/`, and the one machine-local `local/` directory. | [ADR-0008](../cocoder/decisions/0008-repository-topology.md) |
| Workspace | A managed project registered with CoCoder. A workspace has one primary root and may have secondary roots. | [ADR-0019](../cocoder/decisions/0019-multi-root-workspaces.md), [ADR-0027](../cocoder/decisions/0027-workspace-storage-contract.md) |
| Primary root | The workspace root that owns the tracked `cocoder/` governance directory and defines the workspace's identity. | [ADR-0008](../cocoder/decisions/0008-repository-topology.md), [ADR-0019](../cocoder/decisions/0019-multi-root-workspaces.md) |
| Secondary root | A non-primary workspace root registered alongside the primary root. A secondary root can be writable or read-only, but it does not own the workspace's `cocoder/` governance directory. | [ADR-0019](../cocoder/decisions/0019-multi-root-workspaces.md) |
| Local zone | Machine-local, gitignored install-private state under `local/`, including the operational DB, run artifacts, worktrees, settings, secrets, and workspace definition files. | [ARCHITECTURE.md](../ARCHITECTURE.md) |
| Governance | Tracked workspace control files under `cocoder/`: priorities, decisions, tickets, memory, persona extensions, standards extensions, and related workspace policy. | [ADR-0003](../cocoder/decisions/0003-data-model-hybrid.md), [ADR-0008](../cocoder/decisions/0008-repository-topology.md) |
| Dogfood | CoCoder's own managed workspace, hosted inside the CoCoder repo. Its primary root is the install root, and its governance lives at `CoCoder/cocoder/`. | [ARCHITECTURE.md](../ARCHITECTURE.md) |
| Priority | A launchable, founder-approved objective. In the live workspace, active priorities are flat markdown files in `cocoder/priorities/*.md`. All priorities are Playbooks, but not all Playbooks are priorities. | [ADR-0010](../cocoder/decisions/0010-taxonomy-and-authoring.md), [cocoder/priorities/AGENTS.md](../cocoder/priorities/AGENTS.md) |
| Playbook | A self-tracking plan that can be run over multiple sessions. A priority is one launchable Playbook form; other Playbooks can exist as procedural or planning artifacts outside the active priority set. | [cocoder/priorities/AGENTS.md](../cocoder/priorities/AGENTS.md), [ADR-0032](../cocoder/decisions/0032-retire-playbooks-genre.md) |
| Ticket | A tracked bug, task, or question under `cocoder/tickets/`. Tickets are separate from priorities: they can be launched through ticket-fix runs or linked to a priority, but they are not themselves priority Playbooks. | [cocoder/tickets/AGENTS.md](../cocoder/tickets/AGENTS.md) |
| Run | One execution attempt against a priority, ticket, or ad-hoc task. Runs create operational state, events, sessions, work items, artifacts, and commit links. | [ADR-0003](../cocoder/decisions/0003-data-model-hybrid.md) |
| Display run label | The per-workspace founder-facing handle for a run, formatted as `[workspace] run N` (e.g. `CoCoder run 98`). Primary label in the dashboard Runs tab, Oz chat/awareness, Deb status, and wrap delivery. | [packages/core/src/store/portable/display.ts](../packages/core/src/store/portable/display.ts) |
| Technical run id | The internal runner DB identifier (`run_<n>`). Recorded in durable artifacts (commit messages, `run.json`, ledgers) as a secondary parenthetical; not the primary conversational handle on founder-facing surfaces. | [packages/core/src/store/portable/display.ts](../packages/core/src/store/portable/display.ts) |
| Runner | The core orchestration loop that launches personas, polls directive/verify artifacts, monitors work, applies the verify gate, commits through the commit spine, writes run records, and stops or wraps the run. | [packages/core/src/runner/runner.ts](../packages/core/src/runner/runner.ts) |
| Runner impact | Launch-time detection that a priority would impair the machinery needed to run it — run-critical scope, destructive flag, or `independent-of-runner` marking. Refuses silent dogfooding through the normal daemon runner. | [packages/core/src/priorities/runner-impact.ts](../packages/core/src/priorities/runner-impact.ts) |
| `independent-of-runner` | Priority frontmatter marker: the priority must execute via the runnerless path, not the deterministic daemon runner. | [cocoder/priorities/runnerless-independent-priority.md](../cocoder/priorities/runnerless-independent-priority.md) |
| Runnerless path | `cocoder run-independent <priorityId>` runs an `independent-of-runner` priority by reusing the proven runner loop standalone outside the daemon: frozen installed code, no daemon scheduling/commit spine, direct-git commits, scratch-store isolation for `destructive` priorities, and a fail-fast guard (or `--force` warning) when the daemon is live and the run would open the shared live store. Oscar's model resolves to the first `listModels()` entry (most-capable-by-curation), overriding any pinned assignment on this path. | [cocoder/priorities/runnerless-independent-priority.md](../cocoder/priorities/runnerless-independent-priority.md) |
| Atom | One delegated unit of work inside a run, assigned by Oscar to Bob and verified before commit. | [ADR-0013](../cocoder/decisions/0013-orchestration-observation.md) |
| Persona | A role on the AI team, backed by a prompt plus a CLI/model assignment. Base personas ship with CoCoder; workspace `cocoder/personas/` files extend or add local personas. | [ADR-0005](../cocoder/decisions/0005-personas-and-subtasks.md), [ADR-0012](../cocoder/decisions/0012-living-base-personas.md) |
| Oz | The cross-workspace overseer persona and dashboard chat surface. Oz helps the founder inspect status and route launch, stop, teardown, repair, authoring, and support-commit actions through sanctioned control paths. | [ADR-0017](../cocoder/decisions/0017-oz-orchestration-persona.md) |
| Oscar | The orchestrator persona for a workspace run. Oscar scopes atoms, delegates to Bob, verifies the actual diff/evidence, decides whether to continue or wrap, and never builds normal product atoms himself. | [ADR-0005](../cocoder/decisions/0005-personas-and-subtasks.md), [ADR-0013](../cocoder/decisions/0013-orchestration-observation.md) |
| Bob | The builder/architect persona. Bob implements delegated atoms inside the run's write scope and reports completion for Oscar to verify. | [ADR-0005](../cocoder/decisions/0005-personas-and-subtasks.md) |
| Deb | The escalation/debugger persona. Deb observes run health through runner-owned status feeds, triages faults, recommends narrow Oscar nudges, and repairs CoCoder-owned machinery when in scope. | [ADR-0016](../cocoder/decisions/0016-deb-scoped-repair-fallback.md) |
| Quinn | The user-interaction QA persona. Quinn is reserved for browser/app automation and experience verification. | [ADR-0005](../cocoder/decisions/0005-personas-and-subtasks.md) |
| CLI | A local agent command-line tool that CoCoder drives through an adapter, such as Claude Code, Codex, Cursor, Grok, or Gemini. | [ADR-0006](../cocoder/decisions/0006-adapter-contract.md) |
| Model | The model selection used by a CLI assignment. Personas and per-(persona, Play) bindings choose a CLI plus a model; an empty or default model means the CLI's configured default. | [ADR-0005](../cocoder/decisions/0005-personas-and-subtasks.md), [ADR-0006](../cocoder/decisions/0006-adapter-contract.md) |
| Adapter | CoCoder's driver for a CLI: it builds commands, runs preflight checks, probes capability, and normalizes CLI execution for the runner or daemon. | [ADR-0006](../cocoder/decisions/0006-adapter-contract.md) |
| Play | A CoCoder workflow/procedure with declared trigger class, execution model, write scope, and optional validation. Plays are CoCoder orchestration artifacts, not model-host features. | [ADR-0010](../cocoder/decisions/0010-taxonomy-and-authoring.md), [ARCHITECTURE.md](../ARCHITECTURE.md#play-system) |
| Write scope | The path allow-list a persona or Play is expected to modify. In the current commit spine, scope is advisory for visibility: changed files are committed and out-of-lane paths are flagged, not silently withheld. | [ADR-0007](../cocoder/decisions/0007-write-scope-enforcement.md), [ADR-0023](../cocoder/decisions/0023-workspace-commit-spine.md) |
| Commit spine | The single path for tracked changes to become commits, used by verified atoms, support edits, repair commits, authoring Plays, and daemon mutations. | [ADR-0023](../cocoder/decisions/0023-workspace-commit-spine.md) |
| Commit gate | The runner/core mechanism that records changed files, applies write-scope visibility, creates commit links/events, and produces a durable receipt. | [packages/core/src/commit-gate](../packages/core/src/commit-gate) |
| Directive | A runner-polled file written by Oscar, usually `directive-N.json`, telling the runner to delegate the next atom or wrap up. | [packages/core/src/runner/directive.ts](../packages/core/src/runner/directive.ts) |
| Verify gate | Oscar's required pass/fail verdict before an atom commit. A passing verdict commits the atom; a failing verdict quarantines the atom's changes and asks for the next directive. | [ADR-0013](../cocoder/decisions/0013-orchestration-observation.md) |
| Wrap-up | The run-ending handoff after Oscar decides there is a real stop condition. The runner validates and delivers the closeout/pickup and final landing outcome. | [packages/personas/base/plays/wrap-up.md](../packages/personas/base/plays/wrap-up.md) |
| Pickup | A resumable brief written at wrap-up so a future session can continue without rereading the entire run. | [ADR-0002](../cocoder/decisions/0002-operational-loop.md) |
| Teardown | The explicit run lifecycle action that closes a run's live sessions/panes after the founder asks for it. Teardown is process/window cleanup, not a persona-authored code or governance edit. | [ADR-0017](../cocoder/decisions/0017-oz-orchestration-persona.md) |
| ADR / Decision | A durable architecture or governance decision under `cocoder/decisions/`. Accepted ADRs are the historical owner for why a rule exists; current-truth docs summarize them and link back. | [cocoder/decisions/README.md](../cocoder/decisions/README.md) |
| Memory | Durable workspace knowledge under `cocoder/memory/` that should outlive one run and be read directly by future sessions. Memory files are canonical for their own content and are not mirrored through an index. | [cocoder/memory/AGENTS.md](../cocoder/memory/AGENTS.md) |
| Elegance / Elegant | The third rank of CoCoder's quality order — **correctness first, clarity second, elegance third**. Maximum effect with minimum surface area: the fewest concepts, words, files, knobs, and special cases that still keep behavior, evidence, reversibility, and safeguards. Cashes out as: one owner per concept; remove what carries no weight; order work so the next agent can run it; prefer the boring path. | [shared-standards.md](../packages/personas/base/shared-standards.md) (Elegance Standard) |
| Oz dashboard | The local browser control surface, also called the dashboard or CoCoder dashboard. It talks to the Oz daemon and lets the founder inspect workspaces, CLIs, personas, Plays, priorities, tickets, and runs. | [docs/oz.md](./oz.md), [ADR-0017](../cocoder/decisions/0017-oz-orchestration-persona.md) |
| Oz daemon | The local server behind the dashboard and CLI control commands. It owns live run launch/stop/teardown endpoints and bridges UI actions into core runner calls. | [ADR-0004](../cocoder/decisions/0004-process-architecture.md), [packages/daemon](../packages/daemon) |
